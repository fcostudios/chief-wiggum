# Track D: Playwright E2E Tests (CHI-158 through CHI-162) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add ~45 Playwright E2E tests across 5 feature areas that currently have zero E2E coverage: file explorer, project actions, settings/theme/i18n, permission flows, and split panes/session lifecycle/onboarding.

**Architecture:** All tests run against the real Tauri dev server at `http://localhost:1420` using Playwright's Chromium project. Tests use the existing custom fixture (`tests/e2e/fixtures/app.ts`) which auto-navigates, waits for the grain overlay, and dismisses onboarding. Each CHI issue maps to a new test directory with 2-4 spec files. Tests use semantic selectors (`getByRole`, `getByText`, `getByLabel`) and the established graceful-fallback pattern for environment differences (CLI present/absent).

**Tech Stack:** Playwright 1.x, TypeScript, custom `test`/`expect`/`modKey` fixture from `tests/e2e/fixtures/app.ts`

**Existing patterns to follow:**
- Import `{ test, expect, modKey }` from `../fixtures/app` (relative path)
- Use `test.describe()` blocks to group related scenarios
- Use `page.keyboard.press(\`${modKey}+key\`)` for shortcuts
- Use `page.waitForTimeout(200-250)` after animations/transitions
- Use `.isVisible().catch(() => false)` for graceful environment fallbacks
- Use `getByRole('dialog', { name: '...' })` for modals
- Use `getByRole('button', { name: '...' })` for buttons with accessible names

```typescript
// tests/e2e/explorer/file-tree.spec.ts
import { test, expect, modKey } from '../fixtures/app';

test.describe('File Tree (CHI-158)', () => {
  test.beforeEach(async ({ page }) => {
    // Switch to Files view via sidebar
    const filesToggle = page.getByLabel('Toggle files');
    const filesButton = page.getByText('Files', { exact: true }).first();
    const toggle = await filesToggle.isVisible().catch(() => false)
      ? filesToggle
      : filesButton;

    if (await toggle.isVisible().catch(() => false)) {
      await toggle.click();
      await page.waitForTimeout(250);
    }
  });

  test('file tree renders when Files section is open', async ({ page }) => {
    const tree = page.getByRole('tree', { name: 'File explorer' });
    const isTreeVisible = await tree.isVisible().catch(() => false);

    if (!isTreeVisible) {
      // No project loaded — check for empty/no-files state
      const noFiles = page.getByText(/No files|No project/i);
      const noFilesVisible = await noFiles.isVisible().catch(() => false);
      if (noFilesVisible) {
        await expect(noFiles).toBeVisible();
        return;
      }
      // Tree might still be loading
      await expect(tree).toBeVisible({ timeout: 10_000 });
    }

    await expect(tree).toBeVisible();
    // Tree should contain at least one treeitem
    const items = page.getByRole('treeitem');
    await expect(items.first()).toBeVisible({ timeout: 5_000 });
  });

  test('search input filters file tree', async ({ page }) => {
    const searchInput = page.getByPlaceholder(/Search files/i);
    const isSearchVisible = await searchInput.isVisible().catch(() => false);
    if (!isSearchVisible) return; // No project loaded

    await searchInput.fill('package');
    await page.waitForTimeout(300); // Debounce delay

    // Should show filtered results or "No files found"
    const noResults = page.getByText(/No files found/i);
    const treeItems = page.getByRole('treeitem');

    const hasResults = await treeItems.first().isVisible().catch(() => false);
    const hasNoResults = await noResults.isVisible().catch(() => false);

    expect(hasResults || hasNoResults).toBe(true);

    // Clear search
    await searchInput.clear();
  });

  test('folder nodes expand and collapse on click', async ({ page }) => {
    const tree = page.getByRole('tree', { name: 'File explorer' });
    if (!(await tree.isVisible().catch(() => false))) return;

    // Find first folder (treeitem with aria-expanded)
    const folders = page.locator('[role="treeitem"][aria-expanded]');
    const folderCount = await folders.count();
    if (folderCount === 0) return; // No folders in tree

    const firstFolder = folders.first();
    const initialExpanded = await firstFolder.getAttribute('aria-expanded');

    await firstFolder.click();
    await page.waitForTimeout(200);

    const afterClickExpanded = await firstFolder.getAttribute('aria-expanded');
    expect(afterClickExpanded).not.toBe(initialExpanded);

    // Click again to toggle back
    await firstFolder.click();
    await page.waitForTimeout(200);

    const afterToggleBack = await firstFolder.getAttribute('aria-expanded');
    expect(afterToggleBack).toBe(initialExpanded);
  });
});
```

```typescript
import { test, expect, modKey } from '../fixtures/app';

test.describe('Theme Settings (CHI-160)', () => {
  test.beforeEach(async ({ page }) => {
    await page.keyboard.press(`${modKey}+,`);
    await page.waitForTimeout(300);
  });

  test.afterEach(async ({ page }) => {
    // Close settings if open
    const dialog = page.getByRole('dialog', { name: 'Settings' });
    if (await dialog.isVisible().catch(() => false)) {
      await page.keyboard.press('Escape');
    }
  });

  test('theme selector shows Dark/Light/System options', async ({ page }) => {
    const dialog = page.getByRole('dialog', { name: 'Settings' });

    // Find the Theme select control
    const themeSelect = dialog.locator('select[aria-label="Theme"]');
    const isVisible = await themeSelect.isVisible().catch(() => false);

    if (!isVisible) {
      // May need to navigate to the appearance category first
      const appearanceButton = dialog.getByText(/Appearance/i);
      if (await appearanceButton.isVisible().catch(() => false)) {
        await appearanceButton.click();
        await page.waitForTimeout(200);
      }
    }

    const themeSelectFinal = dialog.locator('select[aria-label="Theme"]');
    if (!(await themeSelectFinal.isVisible().catch(() => false))) return;

    // Verify options exist
    const options = themeSelectFinal.locator('option');
    const optionTexts = await options.allTextContents();
    expect(optionTexts.some((t) => /dark/i.test(t))).toBe(true);
    expect(optionTexts.some((t) => /light/i.test(t))).toBe(true);
    expect(optionTexts.some((t) => /system/i.test(t))).toBe(true);
  });

  test('switching theme changes CSS variables', async ({ page }) => {
    const dialog = page.getByRole('dialog', { name: 'Settings' });

    // Navigate to appearance if needed
    const appearanceButton = dialog.getByText(/Appearance/i);
    if (await appearanceButton.isVisible().catch(() => false)) {
      await appearanceButton.click();
      await page.waitForTimeout(200);
    }

    const themeSelect = dialog.locator('select[aria-label="Theme"]');
    if (!(await themeSelect.isVisible().catch(() => false))) return;

    // Get current bg color
    const bgBefore = await page.evaluate(() =>
      getComputedStyle(document.documentElement).getPropertyValue('--color-bg-primary').trim(),
    );

    // Switch to Light
    await themeSelect.selectOption('light');
    await page.waitForTimeout(500);

    const bgAfterLight = await page.evaluate(() =>
      getComputedStyle(document.documentElement).getPropertyValue('--color-bg-primary').trim(),
    );

    // If it was already light, bg won't change — but if it was dark, it should
    // Switch back to Dark
    await themeSelect.selectOption('dark');
    await page.waitForTimeout(500);
```

## Architecture Notes for Implementer

### Fixture Usage
- All tests except onboarding use `import { test, expect, modKey } from '../fixtures/app'`
- Onboarding test overrides the fixture to skip auto-dismiss: `import { test as rawTest } from '@playwright/test'`

### Graceful Fallback Pattern
Every test that depends on environment state (project loaded, CLI available, actions present) uses:
```typescript
const isVisible = await element.isVisible().catch(() => false);
if (!isVisible) return; // Graceful skip
```
This ensures tests pass in CI where the full Tauri backend may not have a project open.

### Cleanup Pattern
Tests that modify state (YOLO mode, developer mode, settings values) use `test.afterEach` to restore original state, preventing test pollution.

### Selector Priority
1. `getByRole('dialog', { name: '...' })` — modals/dialogs
2. `getByRole('button', { name: '...' })` — buttons with labels
3. `getByLabel('...')` — elements with aria-label
4. `getByPlaceholder('...')` — input fields
5. `getByText('...')` — visible text content
6. `locator('[role="treeitem"]')` — complex accessible elements
7. `locator('select[aria-label="..."]')` — native select elements
